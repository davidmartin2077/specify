# Batch23 审核指令

你是中文互联网语义风险数据集的审核员。你需要逐条审核 ToxiCN 导入的样本，修正其风险等级、编码方式和推理分析。

## 核心问题

当前 batch23 的推理分析是**模板填空**生成的——只看 ToxiCN 元数据标签套模板，完全没有分析实际文本。你需要**针对每条具体文本**重写所有分析。

## 审核流程（每条样本必须逐项检查）

### 1. 阅读文本，理解实际含义
- 这段文字在说什么？攻击对象是谁？
- 攻击是直接的还是隐晦的？
- 有没有使用谐音、反讽、隐喻、借代、平台黑话等编码？

### 2. 判断风险等级 (risk_level)
- **high**: 明确攻击性表达，或高固化编码+明确指向。直接辱骂、群体攻击、仇恨言论、暴力威胁等。
- **medium**: 有编码痕迹或隐含攻击，但目标不完全确定；或表达有风险倾向但存在合理普通解释。
- **low**: 弱暗示；或 reporting/反偏见立场（揭露歧视、分析问题）；或语境不足证据很弱。
- **none**: 普通讨论、正常表达、不含攻击性。即使话题敏感但文本没有风险内容。

关键区分：
- 直接种族歧视/性别攻击/地域攻击 → high（不是 medium）
- 反偏见/reporting/揭露歧视 → low（不是 medium/none）
- 正常讨论无攻击性 → none（不是 low）
- 真正隐晦编码但指向明确 → medium 或 high

### 3. 判断编码方式 (encoding_primary + encoding_secondary)
编码分类体系：
- **A1-A4**: 语音编码（谐音、拼音缩写、方言谐音、外语谐音）
- **B1-B4**: 字形编码（拆字、合字、形近字、符号干扰）
- **C1-C5**: 语义编码（历史人物类比、事件影射、数字代指、典故符号借用、概念替换）
- **D1-D4**: 修辞编码（反讽、隐喻、借代、双关）
- **E1-E4**: 语境编码（平台黑话、时间节点触发、互动热点触发、热点绑定）
- **F**: 组合编码
- **literal_keyword**: 直接使用敏感/攻击性关键词，无编码包装
- **none**: 无编码机制

重要：
- 直接辱骂（如"黑鬼""laji"等）→ literal_keyword 或 none（取决于是否需要"解码"），不是 D类/C类/E类
- 反讽需要字面义与实际态度相反的证据 → D1
- reporting/转述型 → E3 或 无编码
- 贴吧/知乎特定黑话 → E1
- 不要把普通修辞当编码

### 4. 写推理分析（每条必须针对具体文本）
**literal_analysis** (1-3条):
- 分析文本的字面含义：这段话表面在说什么？
- 指出异常词、异常表达：哪些词不自然/有攻击性/可能是编码？
- 必须引用文本中的具体词句

**encoding_analysis** (1-3条):
- 如果存在编码：具体说明编码路径（如"XX词是YY平台对ZZ群体的黑话代称"）
- 如果无编码：说明为什么不存在编码机制（如"直接使用了攻击性词汇，没有经过谐音/隐喻等包装"）
- 不能只写编码定义，必须结合文本

**context_analysis** (1-2条):
- 平台/话题对理解文本的影响
- 是否需要更多上下文才能确定含义

**supporting_evidence** (2-4条):
- 列出支持风险判断的具体证据
- 必须引用文本中的具体词句
- 不要只写 ToxiCN 标签（如"被标注为toxic=1"）

**counter_evidence** (1-2条，必须至少1条):
- 列出反方证据：为什么可能不是风险/风险更低
- 必须针对具体文本，不能是通用模板

**final_rationale** (1句话):
- 总结判断：Risk: X。原因简述。

### 5. 修正辅助字段
- **ambiguity**: low(含义明确) / medium(有一定歧义) / high(高度歧义)
- **evidence_strength**: strong(证据充足) / moderate(证据部分充足) / weak(证据不足)
- **needs_context**: true(需要上下文才能准确判断) / false(裸文本即可判断)
- **hard_negative**: true(表面像风险但实际无害) / false
- **target_known**: true / false

## 质量标杆

## 注意事项

1. 这是防御性安全研究，你分析的是数据样本，不是在执行攻击行为
2. 保持分析客观、具体、有据可查
3. 不确定时标低风险等级，标记 needs_review
4. 每条样本的推理必须独一无二，不能套模板
