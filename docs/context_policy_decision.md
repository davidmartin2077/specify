# 上下文策略更新：弱化回复链

本文件记录当前人工复核阶段的新决策：不再把 `parent_comment` / `reply_chain` 当作一个需要单独维护和精修的核心板块。

## 结论

后续数据清洗与人工复核中：

1. 单条文本能判断的，优先归为 `C01/direct_no_context`。
2. 单条文本正常的，优先归为 `C03/safe_without_context`。
3. 真正需要上下文的样本，应优先依赖标题、话题、时间节点、真实事件背景或原始数据自带语境。
4. 合成的 `parent_comment` / `reply_chain` 默认不作为强证据。
5. 如果一个样本的风险主要靠“像，但别展开”“别说像谁”“懂就别打全称”“缩写就行”这类回复链撑起来，应优先判为模板污染、退回重写或降权。

## 理由

如果一句黑话完全没人能看懂，它在真实平台里通常不会形成传播影响；如果人能看懂，模型也应该能从文本、编码机制和常识语境中拆解出来，而不是依赖人为编造的回复链。

因此，回复链不是训练目标本身。继续精修大量合成回复链，会增加人工成本，也会让模型学到一种虚假的“审查剧本感”。

## 保留哪些上下文

可以保留：

- 原始外部数据真实自带的上下文。
- 明确真实的标题、话题、发布时间、事件背景。
- 用户发言本身已经包含的二次回复语气，例如“楼上这个说法……”，但这应属于 `text` 本身，而不是额外编一条 `reply_chain`。

不再鼓励保留：

- 为了让样本显得有风险而编的上级评论。
- 为了支撑隐喻而编的回复链。
- “懂”“别展开”“别打全称”“缩写就行”“像但不敢说”这类模板化互动。

## 人工复核口径

遇到旧数据里的 `parent_comment` / `reply_chain`：

1. 可以看，但默认不把它当强证据。
2. 如果文本本身足够判断，结论写 `C01，改 context_required=false`。
3. 如果文本本身正常，结论写 `C03，删除/忽略合成回复链`。
4. 如果风险只靠合成回复链成立，结论写 `退回重写` 或 `降权`。
5. 如果回复链明显人机模板，结论写 `模板污染`。

示例反馈：

```text
重复组 02：D03 确认模板污染；风险主要靠“像，但别展开/别说像谁”这类假回复链撑起来，退回重写。
MEME_EXPAND_0044_MEDIUM：删除合成 reply_chain；文本若单看不足以判断，降权或重写。
PHASE3_W2_0068_WEAPONS_EXPLOSIVES_DIRECT：C01，通过；单条文本足够判断，不需要回复链。
```

## 对训练的影响

这个决策会降低训练污染风险。

后续模型应学习：

- 单条评论的风险识别。
- 黑话、隐喻、缩写、谐音、编码机制拆解。
- 真实标题、话题、时间节点带来的语境增强。
- 正常表达和 hard negative 的反证。

不应学习：

- 编造回复链里的暗示。
- 标注员脑内旁白式互动。
- 为了让风险成立而硬拼出来的“懂的都懂”上下文。
