# 双义词盲区审计报告

**日期**: 2026-06-18  
**数据集**: 2500 条 (combined_candidates.jsonl)  
**审计范围**: 150+ 中文互联网常见双义词/黑话

---

## 问题定义

模型从训练数据中学习 **词→标签** 的映射。如果某个词在数据集中只以一面出现（比如"唐诗"只在 hard_negative 中作为《唐诗三百首》出现），模型将无法学到该词的另一面（"唐诗"同音"唐氏"——唐氏综合征侮辱）。生产环境中遇到恶意使用时会漏判。

同理，如果某个词只有 risk 样本（如"躺平"永远被标为风险），模型会将所有含"躺平"的文本都判为风险——包括物理上"累了躺平休息一下"的正常表达。

**核心原则：具有二元性的词，数据集中必须两面对照存在。**

## 🔍 额外发现：标签质量问题

### "开车" 的 risk 样本实际不含网络黑话含义

```
id=001484, risk=high: "白完车就是横，刚才路上一个白完牌直接逆行超车..."
→ 这里的"开车"是字面驾驶，risk 来自地域歧视(白完)，不是"开车"作为色情分享的黑话含义
→ 真正的"开车"(=分享色情资源)在网络黑话中的含义完全缺失
```

### "苹果" id=000307 可能有争议

```
id=000307, risk=none, hn=True: "苹果中国制造，美的亡我中华之心不死..."
→ 虽然标为 hard_negative，但文本有明显的民族主义/反美情绪
→ 标签可能是正确的（商业批评不是安全风险），但处于边界
```

## 方法论

每个需要修复的双义词应遵循对照集构造法：

```
同一词 → 4条样本
  ├── [high] 明确风险语境 ("你个唐诗/唐氏，连这都不会")
  ├── [medium] 隐晦/黑话语境 ("这人有点唐")
  ├── [low] 边界/灰色语境 
  └── [none/hn] 正常无害语境 ("我在读唐诗三百首")
```

已有 hard_negative 的词只需补前 3 条；已有 risk 的词只需补最后 1 条。
