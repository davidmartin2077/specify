# 第二阶段数据扩充计划

## 阶段定位

当前项目仍处于数据集准备初始阶段。现有 330 条正式候选只是第一版骨架，适合验证 schema、SFT 格式和切分流程，但不足以让约 4B 小模型稳定学会识别互联网鉴证梗、互联网黑话、风险词语境、隐喻、反讽、人物代指、历史影射和 hard negative 反证。

第二阶段目标不是立刻训练，而是继续构建更大、更稳、更接近真实评论/弹幕的数据池。所有新增候选默认先进入 `data/raw/`，质量状态为 `needs_revision`，经过人工抽查和脚本校验后再决定是否合并 processed。

## 扩充目标

第二阶段优先把数据规模从 330 条推进到一个更适合 4B SFT 起步的候选池。建议分批进行，每批 50-150 条，先看风格再扩大。

建议阶段性目标：

1. 第一小批：50-100 条，验证 phase2 生成风格。
2. 第二批：累计 500-800 条，覆盖主要风险语境和 hard negative。
3. 第三批：累计 1500-3000 条，开始具备小模型 SFT 的基础覆盖。
4. 后续：根据评测缺口继续补齐具体梗簇、平台黑话、风险词类别和反证场景。

## 能力覆盖轴

### 1. 互联网鉴证梗与人物/历史代指

目标能力：

1. 识别历史人物、影视角色、典故、地名、事件、数字和物品被用作现实隐喻。
2. 区分正常历史/影视/文学讨论与现实影射。
3. 能说明证据来自 text、标题、回复链、时间节点还是评论区共识。

建议扩充：

1. 历史人物类比。
2. 历史事件影射。
3. 数字/日期代指。
4. 物品、食物、地名、动作、称号借代。
5. 正常课程、书评、影视讨论、旅游、游戏设定等 hard negative。

### 2. 互联网黑话与平台规避

目标能力：

1. 识别首字母、谐音、拆字、符号间隔、表情替代、平台黑话。
2. 判断“不能说”“懂的都懂”“看主页”等是否真的形成风险。
3. 避免把普通自嘲、玩梗、饭圈缩写、技术缩写全部误判。

建议扩充：

1. 平台规避表达。
2. 首字母/拼音缩写。
3. 谐音/形近/拆字。
4. 群体共识型黑话。
5. 普通缩写、普通玩梗、正常社区规则讨论 hard negative。

### 3. 风险词语境判断

目标能力：

1. 词库命中只是召回线索，不等于风险结论。
2. 对赌博、刷单、诈骗、违禁交易、低俗、辱骂、暴恐、网络黑产等词，能结合语境判断宣传、科普、新闻、举报、课程、引用、反诈、反赌、维权与真实招揽的区别。
3. 强红线类可以更偏高召回，但仍需要识别“举报/新闻/安全教育”与“发布/招揽/教程/交易”的不同。

建议扩充：

1. 反赌宣传、新闻报道、家人劝阻、课程案例。
2. 反诈宣传、求职避坑、平台公告、维权提醒。
3. 正规招聘、正规电商、正常社群运营 vs 诈骗导流。
4. 安全科普、漏洞防护、警方通报 vs 网络黑产教程。
5. 医学/法律/社会新闻语境中的敏感词撞车。

### 4. 正常表达与 hard negative

目标能力：

1. 不把正常话乱封。
2. 不把所有“懂的都懂”“不能说”“私信”“群”“主页”“赌博”“刷单”等关键词都判成风险。
3. 学会在证据不足时给 low/none，而不是过度脑补。

建议扩充比例：

1. 第二阶段至少 30%-40% 为 `none` 或 `low`。
2. 其中 hard negative 应覆盖每个主要类别。
3. 每条 none 样本必须有具体反证，不能只写“无风险”。

## 建议配额

初始扩充到 1000 条候选时，可参考以下配额：

1. 鉴证梗/历史影射/人物代指：200 条。
2. 平台黑话/规避表达/缩写谐音：180 条。
3. 风险词语境判断：300 条。
4. 赌博/诈骗/广告导流专项：160 条。
5. 低俗/辱骂/违禁/暴恐/网络黑产强风险类：100 条。
6. 纯 hard negative 专项：200 条。

这些配额可以重叠统计，例如“反诈宣传中的刷单”同时属于风险词语境判断和 hard negative。

## 样本质量标准

新增样本必须满足：

1. `text` 像真实被审核评论/弹幕/回复本身，不写成审核员评价。
2. 不机械逐词扩写词库。
3. 不把所有风险都写成“私聊/懂的都懂/看主页/进群”。
4. high 必须有明确证据，不靠单个词脑补。
5. none 必须写清反证。
6. 强红线类少构造宽松 none；如果构造 none，应是举报、新闻、安全教育、课程、明确反对等场景。
7. 每条记录必须保留来源、扩充轴、语义簇、批次和是否入库状态。

## 禁止事项

1. 不要把 Sensitive-lexicon 全量词条直接变成训练样本。
2. 不要重复生成或覆盖已经确认过的 330 条正式候选。
3. 不要把 raw 候选直接标成 approved。
4. 不要为了凑数制造模板腔、审核员腔、过度解释腔。
5. 不要把敏感目标写得过于直接；必要时保持匿名化目标表述。
6. 不要把正常历史、新闻、课程、宣传、求职、商品、医学、法律讨论误写成高风险。

## 第二阶段产物设计

建议新增：

1. `scripts/build_phase2_seed_candidates.py`：第二阶段候选生成入口。
2. `data/raw/phase2_seed_candidates.jsonl`：第一批 phase2 raw 候选。
3. 后续可新增 `data/processed/phase2_candidates_cleaned.jsonl`：人工复核或脚本清洗后再考虑合并。

第一批 phase2 应以 hard negative 和语境依赖风险词为主，因为它们直接服务于“不要把正常话乱封”的核心训练目标。
